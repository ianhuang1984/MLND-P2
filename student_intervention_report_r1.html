4)    Training and Evaluating Models:

A.  Naive Bayes:

      General Application: Naive Bayes classifier is Based on Bayes' theorem, Naive Bayes classifier aims to find the probability that a particular object belongs to certain class given its observed feature values. The adjective naive derives from the assumption that the features in a dataset are mutually independent. It is widely used as in document classification such as spam email filtering and text grouping. Naive Bayes is also applicable to disease diagnosis and treatment process selection.

 

      Strengths: Nave Bayes classifier is easy to implement, computationally efficient, and perform particularly well on relatively small datasets (i.e. student intervention project) compared to other algorithms.

 

      Weaknesses: Strong violations of the independence assumptions and non-linear classification problems can lead to poor performances of naive Bayes classifiers.

 

      Reason for choosing the model: Assume the likelihood of the features is assumed to be Gaussian. Since student data is relatively small and would be used as reference for much bigger student data entries, naive Bayes comes handy in this situation. Being an eager learner, naive Bayes classifier is known to be relatively fast and computationally efficient in classifying new instance when unseen data is being fed.

B.  Decision Tree

      General Application: By transforming the decision making process a tree learning algorithm, more information gain would be acquired to arrive at the conclusion whether a student needs intervention or not. Decision tree algorithms are often used in the scenario of categorical variables, but the same concept applies to numeric variables with a binary question. Decision tree is often seen in manufacturing and production environments to increase productivity and to spot flaws, and a recent development of ID3 algorithm is applied to medical text classification.

Since the feature column in the student data has been preprocessed to binary values and dummy variables has been assigned to categorical values, it would be rather straightforward to understand and interpret how predictions being made since the tree could be visualized. If theres an attribute for the data to be split on during data preparation process, the original set of examples would be split into groups based on the value of the best attribute, and the set of available attributes to further split on would have the best attribute removed from it.

 

      Strengths: Simplicity of tree classification presentation and straightforward logic process. It is also computationally efficient since the cost of prediction using the tree is logarithmic in the number of data points as opposed to those used to train the tree.

 

      Weaknesses: For data including categorical variables with different numbers of levels, information gain in decision trees is biased in favor of those attributes with more levels. For instance, fathers job and mothers job has many attributes compared to others in this project. In addition, decision tree is prone to overfitting since it might lead to a deep and complex tree with many nodes.

 

      Reasons for choosing the model: Good for interpretability since prediction are broken down into a series of questions. It would be easier to identify what specific node the system should raise red flags and school should intervene as early as possible.

C.   Support Vector Machine

      General Application: Support Vector Machine (SVM) classifier defines the distance between decision boundary (i.e. Pass/Fail) and the training data samples that are closest to the boundary, which being called as support vectors. The main object is to maximize the margin of the distance so that when an unseen student data in the intervention project being tested, an accurate prediction could be made. Given its ability to handle data sets with multiple features, SVM classification is ideal in text categorization, especially in cases of low ratio of sample size to dimension, and in image processing such handwriting identification and face recognition.

Source: Building Predictive Model using SVM and R, DnI Institute, September 13 2015

 

      Strengths: Since only support vectors are used to specify the margin, and the model complexity would not depend on dimensionality, thus it is very applicable to data set with large features such as the student project.

 

      Weaknesses: SVM is sensitive to noise as a relatively small number of mislabeled examples can dramatically decrease model performance. However, this could be mitigated by using soft margin approach.

 

      Reason for choosing the model: Based on multi-feature nature of the student data, it would be beneficial to use SVM classification as it would find the either linear or non-linear separable planes.

5)Choosing the Best Model

Among all three models, I chose Support Vector Machine classifier to build the student intervention model. As all models have its advantages, the other two models of Naive Bayes and Decision Tree clearly have more limitations in theory and in terms of performance of F1 score.

 

For Naive Bayes, the condition of using this classifier banks on factors are considered not dependent to each other and priors would be ignored. In practice, the independence assumption is often violated, but nave Bayes classifiers still tend to perform well under this unrealistic assumption. Nevertheless, a great amount of student data is highly correlated with each other and multicollinearity may simply being too important to overlook. For example, fathers job might be highly related to mothers job in real world scenario. Moreover, the data set nature might not be linearly separable so that naive Bayes classifier might be prone to weak performance given its limitations. Naive Bayes classifier has pretty consistent F1 score in training data set, but its F1 score in testing data set does not improve as sample size increases. Its F1 score performs the best in the first 200 data points of 300 training samples. As training samples increase, training time and prediction time of Naive Bayes maintain the same, and this characteristic reflects that Naive Bayes is computationally efficient and applicable to small data set.

 

In the case of Decision Tree classifier, overfitting is clearly its main flaw. It is clearly shown that a perfect F1 score in training set with huge gaps between testing set in all three different sample sizes. This trait represents that the classifier over fits the training set and undermine the performance applying the same decision tree on testing set. One of the several possible causes is that decision tree is biased toward multi-level attributes. As mentioned earlier, job category of father and mother has many attributes compared to others in this student intervention project. This manifests the biggest disadvantage of Decision Tree classifier despite it is very efficient in prediction time.

 

The main point for selecting Support Vector Machine as the classifier is its highest and consistent F1 score in both training and test data sets. Its F1 score for 300 training samples is identical with for the 100 samples and just slightly less than the 200 samples. Although Support Vector Machine takes significantly more resources to train the model and predict the classification, it is reasonable to compromise for a better performance. Moreover, as we foresee the trend of school size becoming smaller, the consideration of costs should be of lower priority since the resources put in for training and prediction would be capped.

 

Moreover, Support Vector Machine can efficiently perform a non-linear classification using kernel trick to implicitly map the inputs into high-dimensional feature spaces. Kernel The depiction below illustrates how kernel transform non-linearly separable data to hyperplane so that margin maximization could be achieved.

Source: Support Vector Machines Explained, Tristan Fletcher, March 1 2009

To summarize, we can imagine the combination of each feature of a student, and the target of whether the student succeed in passing, both being mapped in a hyperplane. Support Vector Machine classifier then try to justify the distance between each decision boundary of whether each particular student passes or fails. The data samples closest to the boundary are called support vectors, and the classifier would maximize the margin of the distance so that it would be easier and increase the accuracy of classifying a new, unseen data entry. Eventually when a new data set like a new school class being tested, a prediction could be made based on which classification boundary the target falls within.

      Tuned Model Performance: F1 score for training data set improved significantly, but not so much for testing data set as limited performance boosting being observed. F1 score for testing set increased from 0.8053 to 0.8163. Training time is high as exhaustive Grid Search is being applied and three parameters of Kernel, C, and gamma are being tuned.

 

      Kernel: Radial Basis Function(RBF) and Sigmoid are two kernel setting being compared and RBF is chosen.  In this project, it is valid that we apply RBF(Gaussian) kernel since the number of observations is larger than number of features.

 

      Parameter Range: A parameter ranges from 0.0001, 0.001, 0.01, 0.1, 1, 10, 100, to 1000 are being tuned for parameter C and gamma.

 

      C: A C value of 10 and gamma of 0.001 are the better choices to enhance the classifier performance. A low C makes the decision surface smooth, while a high C aims at classifying all training examples correctly by giving the model freedom to select more samples as support vectors. In this case, the tuned C value of 10 is a moderate width of margin and avoids the risk of overfitting all the training samples.

 

      Gamma: The gamma parameter defines how far the influence of a single training example reaches, with low values meaning far and high values meaning close. The tuned gamma value of 0.001 is somehow in the lower side of spectrum, thus the train classifier might lose certain capacity in capturing the data complexity should sample sizes increase.
